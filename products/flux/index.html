Target Users

Writers of multiple publications who prefer Markdown, need to manage different writing workflows, and understand the importance of version control.

Pain Point

Each publication (a book, a newsletter, a blog) could have its own writing workflow. A writer working on multiple publications would need to manage multiple such workflows. It's hard to codify those steps, while keeping everything version-controlled.

Proposed Solution

A prompt-based interactive app that allows writer to navigate and switch between multiple publication workflows with ease

Key Benefits

Automation that frees writer from memorizing all the idiosyncratic steps in each publication workflow. Version control that allows writer to switch between various Git repos. Integration with static site generator (Hugo, React, Vue, Svelte)
